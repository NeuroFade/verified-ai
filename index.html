What is Verified AI

On-chain proof that your AI
actually ran — exactly as claimed.

AI systems are making consequential decisions — loan approvals, medical triage, legal analysis — but there's no way to prove which model ran, what input it received, or what output it produced. Without that proof, AI accountability is theater.

The Problem

AI outputs are unverifiable

Any bad actor can claim their output came from GPT-4o, Claude, or any other model. There's no cryptographic link between a model, its input, and its output — until now.

The Solution

Immutable on-chain attestations

Verified AI hashes both the prompt and response, generates a Groth16 zkML proof, and anchors everything permanently to Base mainnet. Permissionless to read. Impossible to forge.

How it works

Three cryptographic commitments

keccak256(prompt)→inputHash
keccak256(response)→outputHash
Groth16(π)→proofHash
↓ stored in AttestationRegistry on Base mainnet ↓

How to Use

From zero to verified in 5 minutes

Install the SDK, attest your first inference, verify on-chain. That's it.

01

Install the SDK or use direct ABI

Add verified-ai-sdk to your project, or integrate directly via ethers.js / viem — no package needed.

install
SDK ethers.js v6 viem 
npm install verified-ai-sdk
npm install ethers    # v6
npm install viem
02

Connect to Base mainnet

You need a Base mainnet wallet. Gas is ~$0.002 per attestation. Sponsored gas via Bankr is also supported.

client
SDK ethers.js v6 viem 
import { VerifiedAI } from 'verified-ai-sdk'

const client = new VerifiedAI({
  network:    'base-mainnet',
  privateKey: process.env.PRIVATE_KEY,
})
import { ethers } from 'ethers'

const provider = new ethers.JsonRpcProvider(
  'https://mainnet.base.org'
)
const signer = new ethers.Wallet(
  process.env.PRIVATE_KEY, provider
)

// AttestationRegistry ABI (minimal)
const REGISTRY = '0x3dBF622ABC705d2Ec0E07EB0fCbb1AbFDe0281eb'
const registry = new ethers.Contract(REGISTRY, [
  'function registerProvider(string,string,bytes32) external',
  'function submitAttestation(bytes32,bytes32,bytes32,bytes32) external returns (bytes32)',
  'function verify(bytes32) external view returns (bool,tuple)',
  'function isVerifiedProvider(address) external view returns (bool)',
], signer)
import { createWalletClient, createPublicClient, http } from 'viem'
import { base } from 'viem/chains'
import { privateKeyToAccount } from 'viem/accounts'

const account = privateKeyToAccount(process.env.PRIVATE_KEY)
const publicClient  = createPublicClient({ chain: base, transport: http() })
const walletClient  = createWalletClient({ account, chain: base, transport: http() })

const REGISTRY = '0x3dBF622ABC705d2Ec0E07EB0fCbb1AbFDe0281eb'

Register as an inference provider

One-time call. Declare your TEE type and signing key — required before submitting attestations.

register.ts
SDK ethers.js v6 viem 
await client.registerProvider({
  name:      'My Inference Node',
  teeType:   'AWS_NITRO',  // or INTEL_TDX | AMD_SEV_SNP
  publicKey: myTEESigningPublicKey,
})
import { ethers } from 'ethers'
const pubKeyHash = ethers.keccak256(myTEEPublicKeyBytes)

const tx = await registry.registerProvider(
  'My Inference Node',
  'AWS_NITRO',
  pubKeyHash
)
await tx.wait()
console.log('Provider registered:', tx.hash)

04

Attest an inference

Hash your model fingerprint + prompt + response, then submit on-chain. Returns a permanent bytes32 attestation ID.

attest.ts
SDK ethers.js v6 viem 
const result = await client.attest({
  model:    'gpt-4o',
  prompt:   userPrompt,
  response: modelResponse,
})

console.log(result.id)      // bytes32 on-chain ID
console.log(result.txHash)  // basescan.org/tx/0x...
import { ethers } from 'ethers'

const modelHash   = ethers.keccak256(ethers.toUtf8Bytes('gpt-4o-weights-v1'))
const inputHash   = ethers.keccak256(ethers.toUtf8Bytes(userPrompt))
const outputHash  = ethers.keccak256(ethers.toUtf8Bytes(modelResponse))
const hardwareId  = ethers.keccak256(myTEEMeasurement)  // PCR0 / MRTD

const tx = await registry.submitAttestation(
  modelHash, inputHash, outputHash, hardwareId
)
const receipt = await tx.wait()

// Parse attestationId from emitted event
const log = receipt.logs[0]
const iface = registry.interface
const parsed = iface.parseLog(log)
const attestationId = parsed.args.attestationId
console.log('Attestation ID:', attestationId)
import { keccak256, toBytes, decodeEventLog } from 'viem'

const modelHash  = keccak256(toBytes('gpt-4o-weights-v1'))
const inputHash  = keccak256(toBytes(userPrompt))
const outputHash = keccak256(toBytes(modelResponse))
const hardwareId = keccak256(myTEEMeasurement)

const hash = await walletClient.writeContract({
  address: REGISTRY,
  abi: registryAbi,
  functionName: 'submitAttestation',
  args: [modelHash, inputHash, outputHash, hardwareId],
})
const receipt = await publicClient.waitForTransactionReceipt({ hash })
const event = decodeEventLog({ abi: registryAbi, data: receipt.logs[0].data, topics: receipt.logs[0].topics })
console.log('Attestation ID:', event.args.attestationId)
05

Verify any attestation

Permissionless read — no private key needed. Works from any environment: Node.js, browser, or another contract.

verify.ts
SDK ethers.js v6 viem 
const readClient = new VerifiedAI({ network: 'base-mainnet' })

const isValid = await readClient.verify(attestationId)
console.log(isValid)            // true

const record = await readClient.getAttestation(attestationId)
console.log(record.modelId)     // 'gpt-4o'
console.log(record.attester)    // 0x... (submitter)
console.log(record.timestamp)   // unix timestamp on Base
// Read-only provider — no wallet required
const provider = new ethers.JsonRpcProvider('https://mainnet.base.org')
const readRegistry = new ethers.Contract(REGISTRY, [
  'function verify(bytes32) view returns (bool valid, tuple(bytes32,bytes32,bytes32,bytes32,address,uint256,bool) att)',
  'function isVerifiedProvider(address) view returns (bool)',
], provider)

const [valid, att] = await readRegistry.verify(attestationId)
console.log('Valid:', valid)
console.log('Provider:', att.provider)
console.log('Timestamp:', new Date(Number(att.timestamp) * 1000).toISOString())
const [valid, att] = await publicClient.readContract({
  address: REGISTRY,
  abi: registryAbi,
  functionName: 'verify',
  args: [attestationId],
})
console.log('Valid:', valid)
console.log('Provider:', att.provider)
console.log('Timestamp:', new Date(Number(att.timestamp) * 1000).toISOString())
06

Submit a ZK proof (optional)

For maximum verifiability, link a Groth16 zkML proof to the attestation. Proof commitment stored on ZKVerifier contract at 0xc303124d…c85d3.

proof.ts
SDK ethers.js v6 viem 
await client.submitProof({
  attestationId: result.id,
  proof:         groth16ProofBytes,  // from Lagrange DeepProve
  publicInputs:  publicInputsHash,
})

const hasProof = await readClient.verifyProof(result.id)
console.log(hasProof)  // true
const ZKVERIFIER = '0xc303124d9276Ea7D3d75E94cE7fE5bd3DBec85d3'
const zkv = new ethers.Contract(ZKVERIFIER, [
  'function submitProof(bytes32,bytes32,bytes32) external returns (bytes32)',
  'function hasValidProof(bytes32) view returns (bool)',
], signer)

const proofHash        = ethers.keccak256(groth16ProofBytes)
const publicInputsHash = ethers.keccak256(ethers.concat([modelHash, outputHash]))

const tx = await zkv.submitProof(attestationId, proofHash, publicInputsHash)
await tx.wait()

const ok = await zkv.hasValidProof(attestationId)
console.log('Proof on-chain:', ok)  // true
const ZKVERIFIER = '0xc303124d9276Ea7D3d75E94cE7fE5bd3DBec85d3'
const proofHash        = keccak256(groth16ProofBytes)
const publicInputsHash = keccak256(concat([modelHash, outputHash]))

const hash = await walletClient.writeContract({
  address: ZKVERIFIER, abi: zkVerifierAbi,
  functionName: 'submitProof',
  args: [attestationId, proofHash, publicInputsHash],
})
await publicClient.waitForTransactionReceipt({ hash })

const ok = await publicClient.readContract({
  address: ZKVERIFIER, abi: zkVerifierAbi,
  functionName: 'hasValidProof', args: [attestationId],
})
console.log('Proof on-chain:', ok)

Architecture

End-to-end verification

From provider registration to on-chain proof — five composable steps.

01
🪪

Register

Provider registers model, TEE type + endpoint on AttestationRegistry

02
⚙️

Infer

Model runs inference off-chain, captures execution metadata

03
🔐

Prove

zkML proof generated via Lagrange DeepProve — ~256 bytes

04
⛓️

Attest

Proof commitment submitted to ZKVerifier, attestation stored on-chain

05
✅

Verify

Anyone verifies trustlessly — permissionless, no third party
